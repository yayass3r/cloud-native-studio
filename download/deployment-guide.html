1.2 متطلبات المتصفح (حرجة)
تنبيه مهم: WebContainers — وهو المحرك الأساسي للمشروع — يعمل فقط عبر HTTPS مع رؤوس COOP/COEP. بدون هذه المتطلبات لن تعمل بيئة التطوير داخل المتصفح.
المتصفح الإصدار الأدنى الحالة ملاحظة
Chrome / Edge 89+ يعمل بالكامل الأفضل للتجربة
Firefox 108+ يعمل بقيود قد تكون بعض الميزات محدودة
Safari 16.4+ غير مدعوم SharedArrayBuffer غير متوفر
متصفح الهاتف — غير مدعوم WebContainers لا يعمل على الموبايل
عن واجهة الهاتف: المشروع يحتوي على واجهة هاتف متكاملة (تصميم متجاوب + شريط تنقل سفلي) لكن بيئة WebContainers لا تعمل على متصفحات الهاتف. واجهة الهاتف ستعمل بصريًا لكن التطوير بالكامل يحتاج سطح مكتب.

2
إعداد Supabase (قاعدة البيانات)

Supabase يوفر قاعدة بيانات PostgreSQL مجانية بسعة 500MB مع 50,000 صف شهريًا — أكثر من كافٍ لمشروع Cloud-Native Studio. تتضمن الخطة المجانية أيضًا مصادقة المستخدمين والتخزين.

2.1 إنشاء المشروع
1
انتقل إلى supabase.com وأنشئ حسابًا مجانيًا باستخدام GitHub أو البريد الإلكتروني. بعد تسجيل الدخول، انقر على "New Project".
2
املأ بيانات المشروع: اختر اسمًا (مثل cloud-native-studio)، اختر كلمة مرور قوية لقاعدة البيانات، واختر أقرب منطقة لك (مثل eu-west-1 أو us-east-1).
3
انتظر إنشاء المشروع (يستغرق حوالي دقيقتين). بعدها ستصل إلى لوحة التحكم.
2.2 إنشاء الجداول
4
انتقل إلى SQL Editor من القائمة الجانبية. الصق SQL التالي وانقر "Run" لإنشاء جميع الجداول:
-- إنشاء جدول المستخدمين CREATE TABLE users ( id TEXT PRIMARY KEY DEFAULT gen_random_uuid(), email TEXT UNIQUE NOT NULL, name TEXT, avatar TEXT, provider TEXT DEFAULT 'credentials', provider_id TEXT UNIQUE, created_at TIMESTAMPTZ DEFAULT now(), updated_at TIMESTAMPTZ DEFAULT now() ); -- إنشاء جدول المشاريع CREATE TABLE projects ( id TEXT PRIMARY KEY DEFAULT gen_random_uuid(), name TEXT NOT NULL, description TEXT, is_public BOOLEAN DEFAULT false, language TEXT DEFAULT 'javascript', user_id TEXT REFERENCES users(id) ON DELETE CASCADE, created_at TIMESTAMPTZ DEFAULT now(), updated_at TIMESTAMPTZ DEFAULT now() ); -- إنشاء جدول ملفات المشروع CREATE TABLE project_files ( id TEXT PRIMARY KEY DEFAULT gen_random_uuid(), path TEXT NOT NULL, content TEXT DEFAULT '', project_id TEXT REFERENCES projects(id) ON DELETE CASCADE, created_at TIMESTAMPTZ DEFAULT now(), updated_at TIMESTAMPTZ DEFAULT now(), UNIQUE(project_id, path) ); -- إنشاء جدول النسخ الاحتياطية CREATE TABLE snapshots ( id TEXT PRIMARY KEY DEFAULT gen_random_uuid(), name TEXT NOT NULL, project_id TEXT REFERENCES projects(id) ON DELETE CASCADE, data TEXT NOT NULL, created_at TIMESTAMPTZ DEFAULT now() ); -- إنشاء جدول سجل المحادثات CREATE TABLE chat_histories ( id TEXT PRIMARY KEY DEFAULT gen_random_uuid(), user_id TEXT REFERENCES users(id) ON DELETE CASCADE, project_id TEXT, role TEXT NOT NULL, content TEXT NOT NULL, created_at TIMESTAMPTZ DEFAULT now() ); -- إنشاء الفهارس CREATE INDEX idx_chat_user ON chat_histories(user_id); CREATE INDEX idx_chat_date ON chat_histories(created_at); -- تفعيل RLS (Row Level Security) ALTER TABLE projects ENABLE ROW LEVEL SECURITY; ALTER TABLE project_files ENABLE ROW LEVEL SECURITY; ALTER TABLE snapshots ENABLE ROW LEVEL SECURITY; ALTER TABLE chat_histories ENABLE ROW LEVEL SECURITY;
ما هو RLS؟ Row Level Security يحمي بياناتك. مع تفعيله، لا يمكن لأي مستخدم الوصول إلى بيانات مستخدم آخر. يمكنك لاحقًا إضافة سياسات (Policies) لتحديد من يستطيع قراءة وكتابة كل جدول.

3
إعداد متغيرات البيئة
3.1 الحصول على مفاتيح Supabase
1
من لوحة تحكم Supabase، انتقل إلى Settings → API. ستجد ثلاث قيم تحتاجها:
المتغير الموقع في Supabase الوصف
NEXT_PUBLIC_SUPABASE_URL Project URL رابط المشروع (مثل https://xxx.supabase.co)
NEXT_PUBLIC_SUPABASE_ANON_KEY anon public مفتاح عام — آمن للاستخدام في الواجهة
SUPABASE_SERVICE_ROLE_KEY service_role secret مفتاح خادم — صلاحيات كاملة، لا تشاركه أبدًا
2
لرابط قاعدة البيانات (DATABASE_URL): انتقل إلى Settings → Database → Connection string واختر "URI". استبدل [YOUR-PASSWORD] بكلمة المرور التي اخترتها عند إنشاء المشروع.
3.2 إعداد متغيرات Vercel

بعد ربط المشروع مع Vercel (سنفعل ذلك في الخطوة التالية)، انتقل إلى:

→
Vercel Dashboard → Project → Settings → Environment Variables وأضف المتغيرات التالية واحدة تلو الأخرى:
NEXT_PUBLIC_SUPABASE_URL = https://your-project.supabase.co NEXT_PUBLIC_SUPABASE_ANON_KEY = eyJhbGciOiJIUzI1NiIs... SUPABASE_SERVICE_ROLE_KEY = eyJhbGciOiJIUzI1NiIs... DATABASE_URL = postgresql://postgres:[PASS]@db.xxx.supabase.co:5432/postgres NEXTAUTH_SECRET = [any-random-string-32chars] NEXTAUTH_URL = https://your-domain.vercel.app
ملاحظة عن NEXTAUTH_SECRET: أنشئ سلسلة عشوائية بتشغيل هذا الأمر في الطرفية: openssl rand -base64 32

4
إعداد Vercel (الاستضافة)

Vercel يوفر استضافة مجانية لـ Next.js مع SSL تلقائي وCDN عالمي. الخطة المجانية تتضمن: نطاق فرعي (.vercel.app)، 100GB نقل بيانات شهريًا، ونشر تلقائي من GitHub.

4.1 ربط المشروع
1
سجل الدخول إلى vercel.com باستخدام حساب GitHub الخاص بك. هذا يسهل ربط المستودع مباشرة.
2
انقر "Add New → Project" ثم اختر مستودع yayass3r/cloud-native-studio من القائمة. إذا لم يظهر، انقر "Adjust GitHub App Permissions" وامنح الوصول.
3
إعدادات البناء: اترك الإعدادات الافتراضية كما هي. Vercel يكتشف تلقائيًا أن المشروع Next.js:
Framework Preset: Next.js Build Command: npm run build Output Directory: (تلقائي) Install Command: npm install
رؤوس COOP/COEP: ملف vercel.json الموجود في المشروع يتضمن إعدادات الرؤوس المطلوبة لـ WebContainers. Vercel سيطبقها تلقائيًا عند النشر.
4
انقر "Deploy" وانتظر. يستغرق البناء حوالي 1-2 دقيقة. بعد الانتهاء ستحصل على رابط مثل: https://cloud-native-studio.vercel.app
4.2 إضافة متغيرات البيئة على Vercel
5
بعد النشر الأول: انتقل إلى Settings → Environment Variables وأضف جميع المتغيرات من القسم 3.2 أعلاه. أضفها لبيئات Production, Preview, Development جميعها.
6
أعد النشر: بعد إضافة المتغيرات، انتقل إلى Deployments وانقر على النقاط الثلاث (...) بجانب آخر نشر ثم اختر "Redeploy".
النشر التلقائي: بعد الربط الأول، كل دفع (push) إلى فرع main في GitHub سيُنشِئ نشرًا جديدًا تلقائيًا على Vercel. لا حاجة لنشر يدوي بعد الآن.

5
إعداد النطاق (Domain)

لديك عدة خيارات مجانية للنطاق. Vercel يوفر نطاقًا فرعيًا مجانيًا تلقائيًا، ويمكنك أيضًا الحصول على نطاق مجاني من الدرجة الأولى.

الخيار 1: النطاق الفرعي (الأسهل)

Vercel يمنحك تلقائيًا نطاقًا مثل:

cloud-native-studio.vercel.app

المميزات: يعمل فورًا، SSL تلقائي، لا حاجة لأي إعداد إضافي.

الخيار 2: eu.org (نطاق مجاني حقيقي)

نطاق من الدرجة الأولى مجاني تمامًا مثل:

cloud-native-studio.eu.org

المميزات: نطاق حقيقي مجاني، يتجدد تلقائيًا. العيوب: الموافقة تستغرق أسابيع.

5.1 ربط نطاق مخصص مع Vercel
1
بعد الحصول على النطاق (سواء من eu.org أو أي جهة أخرى)، انتقل إلى Vercel → Settings → Domains.
2
أضف النطاق واتبع التعليمات لإضافة سجلات DNS. عادةً تحتاج إلى إضافة CNAME يشير إلى cname.vercel-dns.com.
3
انتظر انتشار DNS (من دقائق إلى 48 ساعة). Vercel سيُصدِّر شهادة SSL تلقائيًا لنطاقك.
5.2 خطوات الحصول على نطاق eu.org
تحذير: عملية التسجيل في eu.org يدوية ويستغرق الموافقة من أسبوعين إلى عدة أشهر. إذا كنت بحاجة للنطاق فورًا، استخدم النطاق الفرعي من Vercel.
  1. انتقل إلى nic.eu.org وأنشئ حسابًا
  2. اختر اسم النطاق المتاح (مثل cloudstudio.eu.org)
  3. أضف عنوان خادم DNS الخاص بـ Vercel: ns1.vercel-dns.com و ns2.vercel-dns.com
  4. انتظر الموافقة (تكون يدوية من متطوعين)
  5. بعد الموافقة، أضف النطاق في Vercel Dashboard
5.3 خيارات نطاق أخرى مجانية

التحقق والاختبار
6.1 قائمة التحقق بعد النشر

بعد إكمال جميع الخطوات، تحقق من العناصر التالية:

البنية التحتية
  • الموقع يفتح عبر HTTPS
  • رؤوس COOP/COEP تعمل
  • لا أخطاء في Console
  • الملفات الثابتة تُحمَّل
الوظائف
  • WebContainer يشتغل
  • npm install ينفذ بنجاح
  • npm run dev يشغل الخادم
  • المعاينة الحية تعمل
قاعدة البيانات
  • API routes تستجيب
  • إنشاء مشروع يعمل
  • حفظ الملفات يعمل
  • النسخ الاحتياطي يعمل
الواجهة
  • المحرر (Monaco) يعمل
  • الطرفية (xterm) تعمل
  • مستعرض الملفات يعمل
  • AI Chat يستجيب
6.2 اختبار رؤوس COOP/COEP

افتح المتصفح وانتقل إلى عنوان موقعك. افتح DevTools → Network وانقر على أي طلب. تحقق من وجود الرؤوس التالية في الاستجابة:

cross-origin-opener-policy: same-origin cross-origin-embedder-policy: credentialless
طريقة أسرع: افتح Console في المتصفح ونفّذ: typeof SharedArrayBuffer === 'function' // يجب أن يكون true
6.3 اختبار API Routes
// اختبار قائمة المشاريع curl https://your-domain.vercel.app/api/projects // اختبار إنشاء مشروع جديد curl -X POST https://your-domain.vercel.app/api/projects \ -H "Content-Type: application/json" \ -d '{"name":"Test Project","userId":"test-user"}'

7.5 الترقية المستقبلية

عندما ينمو المشروع، يمكنك الترقية خطوة بخطوة:

  • مصادقة المستخدمين: تفعيل Supabase Auth (GitHub/Google login)
  • نطاق مدفوع: شراء نطاق .com أو .dev (حوالي $10/سنة)
  • Pro Plan: Vercel Pro ($20/شهر) لضمان الأداء
  • Supabase Pro: ($25/شهر) لـ 8GB وقاعدة بيانات أكبر
  • Custom CI/CD: GitHub Actions للبناء والاختبار
